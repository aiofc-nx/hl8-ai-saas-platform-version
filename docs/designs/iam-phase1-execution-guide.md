# IAM Phase 1 执行指引

## 目标概述

Phase 1 聚焦建立 IAM 规范落地所需的基础文档骨架与治理支撑，为后续阶段提供统一的执行准绳与决策依据。本阶段完成后，团队应具备：

- 一份结构完整的《IAM 规范》框架，可承载核心原则、阶段计划、职责矩阵、合规要求与风险策略。
- 与规范配套的治理资产（变更日志、评审模板），确保任何后续迭代均可被追踪与审计。

## 关键输出

| 输出                                                   | 说明                                                                             | 责任人                 |
| ------------------------------------------------------ | -------------------------------------------------------------------------------- | ---------------------- |
| `docs/designs/iam-specification.md` 初版大纲           | 包含背景、原则、阶段计划、职责矩阵、合规检查、风险缓解、变更流程等章节的中文骨架 | 架构负责人             |
| `docs/governance/iam-specification-change-log.md`      | 记录版本号、日期、审批人、变更摘要的初始条目                                     | 治理团队               |
| `docs/governance/iam-specification-review-template.md` | 评审会议议程、核心确认项、结论记录栏位的模板                                     | 架构负责人联合治理团队 |

## 详细步骤

1. **规范大纲编写**
   - 依据 `specs/001-define-iam-spec/spec.md` 与相关设计文档，构建统一的大纲结构。
   - 为每个章节预留内容提示与引用链接（如 IAM v2、CASL 方案、四阶段计划）。
   - 确保全部章节说明使用中文，遵循宪章“代码即文档”原则。

2. **变更日志初始化**
   - 在 `docs/governance/iam-specification-change-log.md` 记录版本 `v0.1.0`（或团队约定的初始版本）。
   - 填写创建日期、编写人、预期评审日期，并预留审批人签字位。
   - 说明日志更新触发条件（例如宪章调整、阶段评审结论、治理检查缺口）。

3. **评审模板准备**
   - 在 `docs/governance/iam-specification-review-template.md` 列出评审会议议程：背景回顾、章节检查、风险与行动项、审批结论。
   - 添加 Constitution Check 对照表，包含宪章八项原则与附加约束。
   - 预留评审决定（通过/驳回）与后续任务指派栏，将模板作为后续评审的标准表单。

## 完成判定

- 规范大纲已提交并通过架构负责人初步审阅。
- 变更日志存在首条记录，包含版本号、日期、责任人信息。
- 评审模板可直接用于后续会议，包含议程、检查项与记录区块。

达成以上条件后，即可进入 Phase 2，以合规化的基础资料支撑后续子域规划与治理流程。\*\*\*
